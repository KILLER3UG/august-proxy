# 0.12.21 — Desktop OpenCode / provider body fix

Shipped for the **Tauri desktop app** (bundled `backend-py` + `frontend/desktop`).

## Fixes

- Stop forwarding `session_id: null` (and other null / August-only fields) on
  OpenAI-compatible and Anthropic upstream bodies. Clears OpenCode Console
  `[400] session_id: Invalid input: expected string, received null` on workbench
  chat and the model Test button.
- API format dropdown labels are leaf paths only: `chat/completions`, `messages`,
  `responses`.

## Known remaining

OpenCode Zen can list models that need different wire endpoints than the
provider’s single `apiFormat` — Test/chat may still **404** for Claude/GPT under
an `openaiChat` Zen provider. See `docs/TROUBLESHOOTING.md` / `AGENTS.md`.
