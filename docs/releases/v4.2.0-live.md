## What's new

Closes another §14 spec gap: a `/api/config/live` endpoint pair + settings UI to pick STT/TTS provider/model/voice without hand-editing `config.json`. Same pattern as v4.1.0-fleet.

### Backend
- **`GET /api/config/live`** — returns the merged live config (defaults + user overrides). Reads `data/config.json → auxiliary.live`.
- **`PUT /api/config/live`** — partial-update; rejects unknown fields and non-string values with 400.
- **`backend-py/app/services/live_config_service.py`** — read/validate/write service that owns defaults + validation.

### Frontend
- **`api-client.ts`** — `getLiveConfig()` + `updateLiveConfig(patch)` + `LiveConfig` type.
- **`sections/workspace/LiveSettingsTab.tsx`** — 5 text inputs (sttProvider / sttModel / ttsProvider / ttsModel / ttsVoice). Empty placeholder = "use browser default" per spec.
- **`WorkspaceModelsSection`** — adds "Live (STT/TTS)" as the **8th subtab** (next to "Model Fleet").

### Default config
```json
{
  "sttProvider": "",
  "sttModel":    "",
  "ttsProvider": "",
  "ttsModel":    "",
  "ttsVoice":    ""
}
```
Empty values mean **"use the browser default"** (Web Speech API / speechSynthesis on Chromium). Setting a provider upgrades the speech path.
