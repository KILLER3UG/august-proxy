# v4.4.1-brain-popup — Floating Brain Popup + Pulse Indicator

> **Released:** 2026-06-30
> **Replaces:** `v4.4.0-brain-modal` (centered modal that covered the chat area)
> **Spec gap closed:** §12 — the v4.4 modal worked but blocked chat. User asked: *"display near the icon and floating there, the user has the choice to expand it"*.
> **Follow-up delivered:** SSE-driven pulse dot on the titlebar icon (animates when unseen brain events arrive).
> **Previous release:** [v4.4.0-brain-modal](./v4.4.0-brain-modal.md)

## What's new

The v4.4 modal opened a centered overlay that covered the chat. Per user feedback, that was the wrong pattern: a desktop productivity app wants **non-modal, anchored, persistent** notifications — closer to Slack's unread badges or a Tauri tooltip than to a Settings dialog.

### Before (v4.4)
- Click 🧠 → modal centered on screen, 720×80vh, backdrop blur
- Closes via Esc / × / backdrop click

### After (v4.4.1)
- Click 🧠 → **floating popup** anchored under the icon, ~520×520, no backdrop
- Click anywhere outside → close (Esc / × also work)
- **Pulse dot** on the icon when unseen events arrive — animated, silent until clicked

## Behavior

| Trigger | Effect |
|---|---|
| Click 🧠 icon | Toggle floating popup (anchored under icon, top-right) |
| Click outside popup | Close |
| Esc | Close |
| New brain event arrives (popup closed) | Pulse dot animates on icon, `unseen` count increments |
| Open popup | Reset pulse + unseen to 0 (you've "seen" the new events) |
| Tab click inside popup | Switches between Activity / Learning / Health (no remount, content swaps in place) |

### Why "pulse on close, reset on open"?

Matches the user's mental model: "show me you've got new brain activity since I last looked." The unseen count survives popup opens/closes — events accumulate while you chat, and the pulse shows them waiting for you next time you look.
