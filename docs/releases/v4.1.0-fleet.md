## What's new

Closes a spec gap: §10 calls for a "Model Fleet" settings subtab in the desktop app so users can map each cognitive role (cortex / cerebellum / hippocampus / prefrontal) to a model without editing `config.json` by hand.

### Backend
- **`GET /api/config/model-fleet`** — returns the merged fleet (defaults + user overrides). Reads `data/config.json → auxiliary.model_fleet`, falling back to hardcoded defaults for unset roles.
- **`PUT /api/config/model-fleet`** — partial-update (pass any subset of the four roles; unknown roles rejected with 400; non-string values rejected with 400). Persists via `config_service.save_config`.
- **`backend-py/app/services/model_fleet_service.py`** — small read/validate/write service that owns the merge + validation logic.

### Frontend
- **`api-client.ts`** — `getModelFleet()` + `updateModelFleet(patch)`.
- **`sections/workspace/ModelFleetTab.tsx`** — a 4-row tab with one model picker per role. Each row has a "Clear (use session model)" button that sets the role back to `''`.
- **`WorkspaceModelsSection`** — adds `Model Fleet` as the **7th subtab** (next to "Background & Reflection", both are cognitive-layer model assignments per spec §10). The Brain icon reuses the existing visual language.

### Default fleet (preserved from v2)
```json
{
  "cortex":      "",
  "cerebellum":  "claude-3-haiku-20240307",
  "hippocampus": "claude-3-haiku-20240307",
  "prefrontal":  "claude-3-5-sonnet-20240620"
}
```
`cortex: ""` = "use the chat session's primary model" (caller's responsibility).
