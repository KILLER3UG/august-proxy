## What's new

### Brain dashboard (Learning + System Health tabs)

A new **Brain** section in the desktop app nav with two tabs:

- **Learning** ‚Äî six cards that surface what the brain has learned: heuristics
  (with `source` badge), recent auto-memories, core facts, delta-engine
  activity, sleep-cycle log, and pending skills (with approve/edit/reject).
- **System Health** ‚Äî per-phase status board for 12 cognitive layers (flag,
  status, last self-check, detail). Red cell = failing layer with the failure
  reason.

Backend exposes:

- `GET /api/brain/learning` ‚Äî rich aggregation of `heuristics`, `auto_memories`,
  `sleep_cycle`, `delta_engine`, `pending_skills`, plus legacy `background_review`.
- `GET /api/brain/health` ‚Äî fans out to 12 selfcheck functions, each returning
  `{status, detail, last_check_at}`.
- `DELETE /api/brain/heuristics/{id}`, `PATCH /api/brain/heuristics/{id}` ‚Äî
  curation through the write queue.
- `POST /api/brain/skills/{name}/approve|reject`, `POST /api/brain/run-consolidation`
  ‚Äî skill genesis + on-demand consolidation.

### `/Exam` ‚Äî preparation skills

A new slash command (`/Exam [topic]`, or `/Exam` after attaching files) that
opens a tutor workflow:

1. **Generate** ‚Äî `POST /api/exam/generate` calls the **Prefrontal** model to
   author N multiple-choice questions. Output is validated (exactly 4 options,
   valid `correct_index`, non-empty rationale); malformed model output is
   rejected.
2. **Banner** ‚Äî `ExamBanner.tsx` shows one question at a time as a persistent
   banner with selectable options, an "Ask the model" input, and a Next button.
3. **Answer** ‚Äî `POST /api/exam/{id}/answer` records the attempt and returns
   `{is_correct, correct_index, rationale}`. The client never sees the
   `correct_index` until they answer.
4. **Help** ‚Äî `POST /api/exam/{id}/help` returns a model-generated explanation
   **without revealing the answer** and **without dismissing the banner**.
5. **Add question** ‚Äî `POST /api/exam/{id}/questions` lets the user request a
   specific question ("add one about X") and the model authors it.

### UX invariants (enforced by tests)

- The model authors and submits every question; no client path writes
  `correct_index`.
- One question per banner (never a list dump).
- Multiple choice and "Ask the model" coexist in the same banner.
- Asking for help does **not** dismiss the banner; help renders in a
  separate modal.
- Banner advances only on explicit user action; help/answer never auto-dismiss.
