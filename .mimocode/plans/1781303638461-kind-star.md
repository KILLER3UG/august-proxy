# Plan: Sanitize MCP Tool Schemas

## Problem
MCP servers can return invalid `inputSchema` (e.g., `[{'type':'number'},{'type':'number'},{'type':'number'}]`) that gets passed to upstream APIs, causing 400 errors.

## Fix

### 1. Add shared `sanitizeToolSchema()` in `backend/services/tools/mcp-client.js`
- Export it for use across the codebase
- Logic:
  - `null`/`undefined`/non-object → `{type: 'object', properties: {}}`
  - Top-level array → `{type: 'object', properties: {}}` (function params must be object-shaped)
  - Missing `type` → set to `'object'`
  - Recursively sanitize each value in `properties`
  - Array values inside properties: if array of `{type: string}` objects → `{type: 'array', items: {oneOf: [...]}}`
  - Otherwise pass through unchanged

### 2. Patch BOTH MCP registration paths in `backend/services/tools/mcp-client.js`
- Line ~148 (URL-based MCP servers): `parameters: sanitizeToolSchema(tool.inputSchema)`
- Line ~222 (stdio-based MCP servers): `parameters: sanitizeToolSchema(tool.inputSchema)`

### 3. Patch `backend/adapters/proxy-tools.js`
- Import `sanitizeToolSchema` from mcp-client.js
- Use in `openAiToAnthropicToolDefinition()` (line ~236): sanitize `tool.function.parameters`
- Use in `anthropicToOpenAiToolDefinition()` (line ~253): sanitize `tool.input_schema`
- The existing `sanitizeAnthropicToolDefinition()` at lines 153-181 drops bad schemas to `{type:'object'}` — our new function is smarter but both should call the shared sanitizer

### 4. Patch `backend/adapters/anthropic.js`
- Line ~454 (`t.input_schema`): use `sanitizeToolSchema(t.input_schema)` instead of raw pass-through

### 5. Add unit tests in `backend/test/sanitize-tool-schema.test.js`
- Raw array schema → `{type:'object', properties:{}}`
- Array-of-type-object property → `{type:'array', items:{oneOf:[...]}}`
- Valid schema pass-through (unchanged)
- Missing type → auto-set to `'object'`
- Nested property sanitization
