# v4.4.4-popup-image-like-resize — Drag-from-Anywhere + 8 Resize Handles

> **Released:** 2026-06-30
> **Spec gap closed:** v4.4.2 had a single SE-corner resize handle and a header-only drag. The user reported *"i can't move it anywhere and i can only adjust the height"*. v4.4.4 makes the popup behave like an image: **drag from any point on the popup**, and resize via **4 corners + 4 edges** (like Photoshop).
> **Bug found during implementation:** v4.4.2's drag test passed by accident — `fireEvent.pointerMove(document, {...})` in RTL attaches props to the React **synthetic** event, but the popup uses **native** document listeners (because it's portaled to `document.body`, where React's synthetic event system doesn't reliably route events). The native listener saw the bare event with no `clientX`, producing NaN positions. Switched tests to dispatch real `MouseEvent`s so props reach the native listeners.
> **Previous release:** [v4.4.3-portal-fix](./v4.4.3-portal-fix.md)

## What's new

### Drag from anywhere
The whole popup is now a drag target (excluding interactive children — buttons/tabs/inputs still get clicks). Cursor: `grab` by default.

### 8 resize handles (image-editor style)
- 4 corners: NW, NE, SE, SW
- 4 edges: N, S, E, W
- "Any drag grows" semantics — drag any direction, the relevant axis (or both) grows. Clicking the popup never accidentally shrinks it.
- Anchored-corner stays put, the opposite edge moves with the cursor.

### Only × button closes
Per user feedback, `Esc` and click-outside no longer dismiss. The × button is the only close action.

## Architecture

```
┌─ BrainIndicator ──────────────────────────────────────────────────┐
│  <button titlebar-brain-button />    (toggle)                      │
│                                                                   │
│  SSE: openBrainEventStream() ──▶ bumps `unseen` on each event    │
│  pulse dot animates when unseen > 0                               │
│                                                                   │
│  useState geom (loaded from localStorage)                          │
│  useRef resizeState / dragState (per-drag state, no re-render)    │
│                                                                   │
│  on pointerdown on handle →                                       │
│    1. resizeState.current = { edge, startX, startY, originX/Y/W/H }│
│    2. document.addEventListener('pointermove', onMove)            │
│    3. document.addEventListener('pointerup', onUp)                │
│  on pointermove → setGeom(prev => grow(axis from drag delta))      │
│  on pointerup → cleanup listeners + persistGeom()                  │
└──────────────────────────────────────────────────────────────────┘
```

### "Any drag grows" math

| Edge | Drag | Effect |
|------|------|--------|
| E (east edge) | any direction | width grows |
| W (west edge) | any direction | width grows, x follows drag |
| S (south edge) | any direction | height grows |
| N (north edge) | any direction | height grows, y follows drag |
| SE corner | any direction | both grow |
| NE corner | any direction | both grow, y follows drag |
| SW corner | any direction | both grow, x follows drag |
| NW corner | any direction | both grow, x + y follow drag |

`clampState` keeps the popup inside the viewport (margin 16px, min 380×320, max viewport - margin).

## Lesson learned (for future overlay work)

**RTL's `fireEvent` attaches properties to the React synthetic event, NOT the underlying native event.** When a component uses **native** listeners (`document.addEventListener`, `window.addEventListener`, or `setPointerCapture`), the native listener sees the bare event with no props. To pass coordinates/etc. to a native listener in tests, dispatch a real `new MouseEvent(type, { ...props })` via `target.dispatchEvent(ev)`.

This bit us in v4.4.2 — the test was passing because the assertion was just `truthy(popup.style.left)`, and the actual left was `NaN` (from `undefined - undefined`). After fixing v4.4.4 to use native dispatch, both tests now meaningfully assert the geometry actually moved.
