## What's new

### `/live` full-window route

A new **Live** section in the desktop app nav (Mic icon, accessible alongside
Chat and Brain).

The LiveSurface composes:

- **LiveOrb** — animated orb reflecting state (`idle` / `listening` / `thinking`
  / `tool-running` / `awaiting-approval` / `speaking`) via framer-motion. A
  `prefers-reduced-motion` variant renders a static ring with the state label.
- **LiveCaptions** — large rolling captions: partial transcript (muted, lower
  opacity) on top of committed transcript (solid). ARIA `aria-live="polite"`.
- **LiveToolRail** — tool activity cards on the left rail. Each card shows
  tool name, args (JSON-stringified), status icon (Loader2 for running,
  CheckCircle2 for done/error). Empty state when no events.
- **LiveApprovalCard** — when `pendingMutations` is non-empty, renders an
  alertdialog with Approve / Deny / voice-confirm (placeholder) buttons.
  ARIA `aria-live="assertive"`.
- **LiveControls** — bottom controls: Mute, push-to-talk/continuous toggle,
  End, Switch to chat. All controls are reachable via keyboard and have
  aria-labels.

### Audio flow

`LiveSurface` wires the full voice pipeline:

```
[start] → STT.start() → onPartial → transcript + state=listening
                          → onFinal → transcript commit + state=thinking
                          → /api/live/turn → TTS.speak(response) → state=speaking
                                                                       ↓ (barge-in)
                                                                  onPartial → TTS.cancel()
```

- STT via `liveSTTFactory()` → Web Speech API when available, provider stub otherwise.
- TTS via `liveTTSFactory()` → Web Speech API when available, provider stub otherwise.
- Cleanup on unmount (`stt.stop()`, `tts.cancel()`, `liveClient.stopSession()`).

### Pluggable STT/TTS interfaces

```ts
export interface LiveSTT {
  start(): Promise<void>;
  stop(): Promise<void>;
  onPartial(cb: (text: string) => void): () => void;
  onFinal(cb: (text: string) => void): () => void;
  onError(cb: (err: Error) => void): () => void;
}
export interface LiveTTS {
  speak(text: string, voice?: string): Promise<void>;
  cancel(): void;
}
```

When real provider integration lands, only `providerSTT.ts` and `providerTTS.ts`
change — UI contracts stay stable.

### Tauri microphone capability

`frontend/desktop/src-tauri/capabilities/default.json` adds `core:audio:default`
and `core:audio:allow-capture`. Existing fs/shell permissions unchanged.
