## What's new

Closes the last big user-facing gap in the Brain dashboard: **the brain now tells you what it's doing in realtime**. When the consolidation daemon merges duplicate heuristics, when the delta engine infers a preference from your edit, when the background review reflects on a session, when skill genesis drafts a new skill вҖ” you see it happen.

### Backend

- **`app/services/brain_event_bus.py`** вҖ” in-process pub/sub. 200-event ring buffer + SSE fan-out (parallel to `services/logger.py:ActivityLog`).
- **`app/routers/brain_activity.py`** вҖ” `GET /api/brain/events?limit=&category=` + `GET /api/brain/events/stream` (SSE).
- **5 publishers** wired into existing subsystems (all defensive: try/except + log + never break the underlying op):
  - `consolidation_daemon.run_consolidation` вҶ’ start + done events (with merge/promote/delete counts)
  - `heuristics_service.add_heuristic` вҶ’ per-rule event with rule_id + source
  - `delta_engine.flush_queue` вҶ’ batch event with local-vs-llm split
  - `background_review._do_review` вҶ’ start + done events (skill recs + fact recs)
  - `consolidation_daemon.{draft,approve,reject}_pending_skill` вҶ’ skill_genesis category

### Frontend

- **`api-client.ts`** вҖ” `getBrainEvents(limit?, category?)` + `openBrainEventStream()` returning a browser `EventSource`.
- **`sections/brain/BrainActivityTab.tsx`** вҖ” feed of `{at, category, summary}` rows, 6 chip filters (All / Consolidation / Heuristics / Delta Engine / Review / Skill Genesis), Pause toggle, auto-merging REST-recent + SSE-live streams (deduped by id).
- **`BrainDashboard.tsx`** вҖ” added **Activity** as the **3rd tab** (between Learning and System Health), with `Activity` icon from lucide-react.
