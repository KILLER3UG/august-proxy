### 2. Model Dropdown Shows Model ID Instead of Name
**Problem**: The dropdown trigger button shows model ID (e.g., `gpt-4o`) instead of friendly display name.
**Current Code** (line 2140):
```tsx
<span className="truncate max-w-[200px] font-medium text-foreground transition-all duration-200">
  {selected ? getModelDisplayName(selected.id || selected.name || '') : 'model'}
</span>
```
**Issue**: `getModelDisplayName()` only strips provider prefix but doesn't prettify. The dropdown list uses `modelDisplayParts()` which does prettify names.

**Fix**: Use `modelDisplayParts()` for the trigger display too, or use `selected.name` if it contains the display name.

**File**: `web/src/sections/chat/ChatThread.tsx` (line ~2140)

### 3. Chat Input Placeholder Shows Model ID

```tsx
placeholder={streaming ? 'August is working…' : (currentModel ? `Message ${getModelDisplayName(currentModel.id)}…` : 'Type a message…')}
```
**Fix**: Use `modelDisplayParts()` or `currentModel.name` for display.

**File**: `web/src/sections/chat/ChatThread.tsx` (line ~1253)

### 4. Model Stops Generating After Many Commands
**Problem**: After many command executions, the model stops generating even though the task isn't done.
**Root Cause Analysis**:
- WebSocket has a 30s ping/pong keepalive (line 191 in chat-ws-service.js)
- If a tool call takes >30s without any streaming, the connection might be considered stale
- The `AbortSignal.timeout(300000)` (5 min) in openai.js is too short for long tasks
- When WebSocket closes without `done`, the frontend shows "Connection lost" (line 747)
- Tool calls that take a long time may cause timeout issues

**Possible Fixes**:
1. Increase the WebSocket ping interval or add activity-based keepalive
2. Increase the fetch timeout in openai.js for long-running requests
3. Add better handling for partial responses when connection drops
4. Add retry logic for tool execution timeouts

**Files**: 
- `backend/services/workbench/chat-ws-service.js` (ping interval)
- `backend/adapters/openai.js` (timeout settings)
- `web/src/sections/chat/ChatThread.tsx` (reconnection logic)
